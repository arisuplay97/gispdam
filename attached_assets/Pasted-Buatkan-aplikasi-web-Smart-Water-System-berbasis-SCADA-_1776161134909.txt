Buatkan aplikasi web Smart Water System berbasis SCADA + WebGIS untuk monitoring dan manajemen distribusi air PDAM, dengan kemampuan digitasi jaringan, import data GIS, dan export ke format standar.

⸻

🎯 TUJUAN SISTEM

Membangun sistem yang:
	•	memonitor kondisi distribusi air (tekanan, status)
	•	menampilkan jaringan pipa dari sumber ke pelanggan
	•	memungkinkan edit/digitasi jaringan langsung di peta
	•	dapat import data dari GIS (QGIS)
	•	menjadi fondasi SCADA penuh (real-time + kontrol)

⸻

🧱 STACK TEKNOLOGI

Frontend:
	•	React.js
	•	Leaflet.js

Backend:
	•	Node.js + Express

Database:
	•	PostgreSQL + PostGIS

Integrasi:
	•	Socket.IO (realtime)
	•	REST API

🗺️ FITUR MAP (WAJIB)
	•	tampilkan base map (Mapbox / ArcGIS)
	•	tampilkan:
	•	marker valve (warna tekanan)
	•	marker sumber
	•	garis pipeline

Warna:
	•	hijau >5 bar
	•	kuning 2–5 bar
	•	merah <2 bar

⸻

✏️ WEBGIS EDIT MODE (FITUR UTAMA)

Mode:
	•	VIEW MODE
	•	EDIT MODE

Di EDIT MODE:
	•	tambah valve (klik map → POINT)
	•	gambar pipa (polyline → LINESTRING)
	•	edit posisi node
	•	hapus data

Gunakan:
	•	Leaflet Draw

⸻

🧲 SNAP FEATURE (PENTING – SEPERTI GAMBAR)

Saat gambar pipa:
	•	otomatis menempel ke node atau garis terdekat

Gunakan:
	•	leaflet-snap plugin

⸻

💾 SIMPAN DATA KE DATABASE

Gunakan PostGIS:

Tabel:

valves:
	•	id
	•	name
	•	pressure
	•	geom (POINT)

pipes:
	•	id
	•	name
	•	diameter
	•	material
	•	geom (LINESTRING)

⸻

🔄 RELASI JARINGAN

Setiap pipa punya:
	•	from_node
	•	to_node

⸻

📥 FITUR IMPORT DARI GIS (WAJIB)

User bisa upload file dari QGIS

Format:
	•	GeoJSON (utama)
	•	Shapefile (opsional)

⸻

Alur Import:
	1.	User upload file GeoJSON
	2.	Backend parse file
	3.	Simpan ke database (PostGIS)
	4.	Tampilkan otomatis di map

⸻

Contoh endpoint:

POST /api/import-geojson

⸻

Contoh parsing:
	•	jika geometry = Point → masuk ke tabel valve
	•	jika geometry = LineString → masuk ke tabel pipe

⸻

📤 FITUR EXPORT

1. Export GeoJSON
	•	semua data (valve + pipe)

2. Export CSV
	•	data valve
